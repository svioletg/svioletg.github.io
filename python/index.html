Installing Python

Most programming languages can't be read and executed by your computer by default, in most cases you'll have to install some software that can take the code you've written, and translate it into something your PC can read. Said software comes in different forms depending on the language in question, but Python uses what's called an interpereter. You can get the latest version of it from https://www.python.org/downloads. As a general note, as of writing this the latest major version is Python 3.12, future versions shouldn't outdate this guide too much, but it could be something to keep in mind. I usually like keeping up with changes to new versions, so if I notice something that could outdate this guide, I'll try and update it when I can.

The yellow download button should automatically give you the installer you need for your operating system. The instructions to follow will be for Windows users, as that's what I run 95% of the time. As far as the Windows installer is concerned, upon opening it you'll want to make sure to check the box next to "Add Python to PATH", or something similar. Frankly, I don't know why this isn't checked by default, but it's required if you want to run Python anywhere on your system. Otherwise, almost nothing I explain beyond this is going to work.

Next, we'll verify that everything installed correctly. Almost every programming language, including Python, needs to be run from a command-line interface in order to work. On Windows, you typically have the Command Prompt and PowerShell, I would suggest using the Command Prompt first. Searching for it in your start menu should bring it up, but alternatively you can also press your Windows key and R at the same time, which will open a "Run" dialog, type "cmd" into the text box, and then either click "OK" or hit Enter, which will also open the Command Prompt.

Using the Command Prompt

If you're not familiar with the Command Prompt or using CLIs in general, I'll give a basic overview here. Here, you can use commands to perform a number of actions on your PC, like moving in and out of folders, managing files, and running scripts and programs. Commands are entirely text-based — in their most basic form, the first word you type is the command you want to run, and anything typed after it are called arguments, which modify the behavior and output of the command. These are separated by spaces, meaning most commands you'll find are either single words, or use underscores (_) or hyphens (-) to subtitute for actual spaces. An example of this syntax would be cd C:\Users\Violet, where cd is the command, and C:\Users\Violet is an argument — in this case, a file path that determines what directory you want to move into. Something to note is that, navigating a modern system, you'll inevitably need to reference paths or programs that contain spaces — in order to make sure the command prompt treats it as one single argument, just put double-quotes " " around what you need to. For example, cd "C:\Program Files".

90% of the time, a "command" is actually just the name of a program somewhere on your PC. Those programs are EXE files, but when using the command prompt, EXEs can be run just by typing their name — no .exe required. When you type something into the command prompt and hit enter, assuming said file isn't already in the directory you're currently residing in, it will search through something called your PATH to locate a program with that name, a term you may remember from the Windows installer.

The PATH is what's called a system enviornment variable — I won't spend a lot of time getting into what that part means, but as for the PATH itself, it's essentially a long list of file paths pointing to different folders — the folders in this list are what your system will search through.

That's how you're able to run certain commands or programs from the command prompt no matter what directory you're currently in, and when you start writing scripts for yourself you may even want to add your own directories to the PATH to have the same convenience. But for now, don't worry about it — the installer should have added all of the relevant Python directories to your PATH already. If it didn't, well, that's what the next section is about to determine.
