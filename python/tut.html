    Introduction

    I'm Violet, a hobbyist programmer who's been learning and using Python since I was a kid. I'm not a professional programmer in any sense and have no intention to be, and I'm probably not qualified to be writing a whole tutorial on this subject. I will be doing this regardless. Python is often listed among good programming languages for beginners, and as I started learning it as a beginner myself all those years ago, I would pretty strongly agree. I think it's a great way to get introduced to the concept of programming in general, and the language makes it fairly easy to jump right into making useful things, so I wanted to write a bit of a tutorial or essay or whatever you want to call it on mainly how to use Python, but also to try and give a better understanding of programming as a whole in the process.

    I want to write this so that it requires as little prerequisite programming or computer knowledge as possible, and I'll be explaining many things as I go, but there are some very common things that are going to come up as we continue that will be useful to get out of the way right now. Firstly, throughout these pages I'll be using the term "CLI" a lot, and the CLI is a very important and basic part to doing almost any coding at all. CLI stands for command-line interface, you're almost guaranteed to have one available to you on any major desktop operating system, like Windows, MacOS, or Linux (any distro of it). If you're on MacOS or a Linux distro, your CLI is likely a program named "Terminal". On Windows 11 you may also have a "Terminal" (or "Windows Terminal") app, which you should use; otherwise, you'll have a program named "Command Prompt" — this exists on basically every version of Windows. Technically Windows also has "PowerShell" as a CLI option, but using it won't provide any advantages as far as this tutorial is concerned—if you already use PowerShell then by all means keep doing so, but if that's the case probably arleady just skipped this section anyways.

    Getting familiar with the CLI

    I'll be using the terms "CLI", "console", and "shell" interchangably, but will mostly stick to "CLI".

    A command-line interface is called as such because it serves solely as an interface to receive and process written "commands" to carry out certain actions. What commands are available to you depends mostly on your operating system, but I'll be covering some basic ones that you'll need for our purposes. Regardless of operating system, you should the cd command available to you. This is used to change your current working directory (directories are the same as folders, the term "directory" is just more common in programming), letting you navigate around your filesystem.

    I'll start by using Windows as an example — when you start your CLI, you should be able to start typing into it. The text to the left of where you're typing is the path to your current working directory, which is likely your user folder (e.g. C:\Users\Violet). The > character at the end is not part of the path, rather it's simply a prompt to show where you can start typing.

    The cd command is very basic and takes only one argument: a path to the directory you want to change into. This can be either relative or absolute. A path is absolute if it starts with the filesystem's root — on Windows, this a drive letter (like C:), and on MacOS or Linux, this is a forward slash (/). Relative paths are any that do not start with that root.

    • C:\Users\Violet\Documents is an absolute path in Windows
    • /home/violet/Documents is an absolute path in MacOS / Linux
    • Documents/Important is a relative path in any OS

    Note that directories are typically separated by backslashes (\) on Windows, but forward slashes (/) on MacOS / Linux. In most cases on modern versions of Windows, the forward slash should be accepted as well, but it's worth keeping in mind in case you're using Windows and run into any trouble regarding paths.

    An example of its usage in Windows...
    C:\Users\Violet>cd Documents C:\Users\Violet\Documents>
    ...and in MacOS / Linux.
    violet@my-desktop:~$ cd Documents violet@my-desktop:~/Documents$

    The other command I'll mention is used to list the contents of your current directory. For Windows, this is the dir command, while on MacOS and Linux, it's ls. Windows...

    C:\Users\Seth>cd Documents C:\Users\Seth\Documents>dir Volume in drive C is Windows Volume Serial Number is XXXX-XXXX Directory of C:\Users\Seth\Documents 2025-09-06 00:00 <DIR> . 2025-09-06 00:00 <DIR> .. 2025-09-06 00:00 48 my_document.txt 1 File(s) 48 bytes 2 Dir(s) XX bytes free C:\Users\Seth\Documents\temptest>
    ...and Linux. Note that the -l argument can be used to give more details such as dates, and the -a argument is used to show hidden files. Both can be done at the same time by using -la.
    violet@my-desktop:~$ cd Documents violet@my-desktop:~/Documents$ ls my_document.txt violet@my-desktop:~/Documents$ ls -l total 4 -rw-rw-r-- 1 violet violet 48 Sep 6 00:00 my_document.txt violet@my-desktop:~/Documents$ ls -a . .. my_document.txt

    With these explained, we can move on. Next up is installing Python itself.

    Setting up Python

    Python code can't be read natively by your computer — in order to run any Python code, you need to install Python's software like the interpreter and standard libraries. If you're on Linux, while Python may be available in your distro's package manager, I would heavily recommend using pyenv to manage your Python installations. For Windows or MacOS users, you can download the latest installer from Python's official site. If you are doing the latter, during the installation you may see a checkbox labelled something like "Add Python to PATH" — it's important to make sure this is checked.

    While there shouldn't be too many differences as long as you're using a version beginning with "3", the latest version as of writing is Python 3.13, so this tutorial is written with that in mind.

    We'll verify that Python was installed correctly. Open up your CLI, type python or py, and you should see something like this:

    Python 3.12.8 (main, Jan 20 2025, 12:56:51) [GCC 11.4.0] on linux Type "help", "copyright", "credits" or "license" for more information. >>>

    When run like this with no arguments, the Python REPL will be opened. The REPL is an interactive console where you can write lines of Python and have them evaluated immediately, letting you experiment and test things out quickly. I won't focus too much on it right now since we haven't even discussed how to write Python yet, but keep it in mind as we continue. In order to actually run Python code, which is saved into files ending in a .py extension, just use the Python command followed by a path to a Python script file.

    python my_code.py
    or
    py my_code.py

    With Python ready to go, we'll start exploring some fundamentals of programming, and how to use them in Python.
